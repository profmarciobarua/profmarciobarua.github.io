Exemplo de Requisito Funcional: Criar Tarefas

Descrição:

O sistema deve permitir que os usuários criem novas tarefas a serem realizadas. Cada tarefa deve conter um título descritivo e uma descrição opcional.

Critérios de Aceitação:

  • O usuário deve ser capaz de acessar a função de criação de tarefas a partir da interface principal.
  • Ao criar uma tarefa, o usuário deve poder fornecer um título e, se desejar, uma descrição mais detalhada.
  • Após criar uma tarefa, ela deve ser exibida na lista de tarefas pendentes.
  • O sistema deve validar se um título foi fornecido para a tarefa; caso contrário, deve exibir uma mensagem de erro.

Exemplo:

Um usuário acessa a interface principal do sistema. Ele clica no botão "Criar Tarefa". Uma tela de criação de tarefas é exibida, onde ele insere o título "Comprar mantimentos" e adiciona a descrição "Leite, ovos, pão". Ele clica em "Salvar" e a tarefa é adicionada à lista de tarefas pendentes com o título e a descrição fornecidos.
