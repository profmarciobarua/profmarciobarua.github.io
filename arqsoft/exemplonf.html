Exemplo de Requisito Não Funcional: Segurança de Dados

Descrição:

O sistema deve garantir a segurança dos dados sensíveis dos usuários, protegendo as informações pessoais e financeiras contra acesso não autorizado e vazamentos.

Critérios de Aceitação:

  • Os dados sensíveis dos usuários devem ser armazenados utilizando métodos de criptografia avançados.
  • O acesso aos dados sensíveis deve ser restrito apenas a usuários autorizados com níveis apropriados de permissão.
  • O sistema deve implementar medidas de prevenção contra ataques comuns, como injeção de SQL e cross-site scripting (XSS).
  • Os logs de atividade devem ser mantidos e monitorados regularmente para detectar atividades suspeitas.

Justificativa:

A segurança dos dados dos usuários é crucial para estabelecer confiança no sistema e evitar possíveis riscos legais e financeiros associados a violações de privacidade.
